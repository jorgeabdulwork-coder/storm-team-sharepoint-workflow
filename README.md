# Pre/Post Storm Request Workflow

**Platform:** FreshService | **Integrations:** Zapier, SharePoint | **Type:** Event-driven automation

## Overview
Built a workflow that automatically classifies and routes storm-related IT service tickets based on request type (Pre-Storm vs Post-Storm) and organizational ownership.

## What It Does
- Triggers on ticket creation when a storm-related Service Catalog Item is detected
- Branches logic based on Pre-Storm vs Post-Storm classification
- Sends the appropriate organizational data to Zapier for downstream automation
- Shares tickets with the correct Operations team for visibility

## Impact
Removed manual triage from storm operations, ensuring requests reach the right team instantly and integrations fire automatically.
